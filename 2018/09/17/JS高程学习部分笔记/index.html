JS高程学习部分笔记

发表于 2018-09-17

1.async异步脚本 ，异步脚本多个之间互不依赖尤为重要，不让页面等待脚本下载执行，从而异步加载其他内容，建议异步脚本不要在加载期间修改dom
2.defer延迟脚本，表示脚本在执行时不会影响页面的构造，也就是说脚本被延迟到整个页面都解析完毕后再运行
3.typeof操作符只能返回undefined boolean string number object function （typeof null === object）
4.IEEE754数值的浮点计算有通病
5.isNaN()适用于对象，会先让对象调用自身的valueOf方法，如果调用后还不能转为number，再调用toString方法，再测试返回值
6.JavaScript对象中具备constructor，hasOwnProperty()方法，isPrototype()，propertyIsEnumerable()，是否可被for in枚举，toString()，valueOf()
7.JavaScript所有函数参数传递都是值传递
8.使用instanceof来检查某个对象是否是某个构造函数的实例， 涉及到原型链
9.执行环境有变量对象与活动对象之分，活动对象不存在于全局环境，每个函数或方法的首个活动对象为arguments对象。
10.执行环境可以保证对变量的有序访问，从局部环境逐步向上访问到全局环境
11.js的垃圾回收机制分两种，标记清除与引用计数，有一个问题需要注意的是循环引用
12.argumens.callee调用自身方法
13.toSting方法传入数字可以改变进制
14.基本类型string具有slice，substr，substring方法，slice与substring的参数都是指定位置，而substr第二个参数是指定个数
15.对象的数据属性的[[Enumberable]]表示能否通过for-in循环返回属性
16.还有[[Configurable]]，[[Writable]],[[Value]]属性
17.通过Object.defineProperty或Object.defineProperties来定义对象的属性描述对象，第一个参数为需要操作的对象
18.读取属性描述对象通过Object.getOwnPropertyDescriptor()，传入对象与属性名
19.工厂模式制造对象

20.构造函数模式，通过使用new操作符

使用new操作符，会经历1.创建一个新的对象2.让构造函数的this指向该对象3.执行构造函数的内部代码4.返回对象
21.原型模式，可共享方法，节约内存开销

22.可用isProtypeOf判断某个对象是否是另个对象的原型
23.in操作符判断某个对象是否可以访问某个属性，即使是在原型上
24.Object.keys, Object.getOwnPropertiesNames, for-in
25.组合使用构造函数模式和原型模式

26.组合继承，借用构造函数与原型继承
27.查询字符串参数

28.location.replace可以跳转页面后禁用后退按钮
